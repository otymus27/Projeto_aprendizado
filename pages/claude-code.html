Claude Code — Manual do Iniciante - CRIADO POR FABIO ROCHA

Tudo que você precisa saber para trabalhar de forma produtiva com o Claude Code: arquivos, comandos, organização e boas práticas.

📁

Arquivos Principais

CLAUDE.md

O arquivo mais importante. Define instruções, contexto do projeto, convenções de código e regras para o Claude seguir durante toda a sessão. Criado na raiz do projeto.

~/.claude/settings.json

Configurações globais do Claude Code. Controla permissões, hooks automáticos, modo de permissão padrão e outras preferências do ambiente.

.claude/settings.json

Configurações locais do projeto (por repositório). Permite sobrescrever as configurações globais para um projeto específico.

MEMORY.md

Índice de memórias persistentes do Claude. Fica em ~/.claude/projects/<projeto>/memory/MEMORY.md. O Claude lê isso no início de cada conversa.

*.md (memórias)

Arquivos individuais de memória (user, feedback, project, reference). Ficam na pasta memory/ e são apontados pelo MEMORY.md.

~/.claude/keybindings.json

Atalhos de teclado personalizados. Permite redefinir teclas como Enter, Shift+Enter, e criar combinações em chord (sequência de teclas).

Comandos Slash

Digite estes comandos diretamente no chat do Claude Code. Alguns são built-in, outros são skills (extensões).

/help
Ajuda Geral Lista todos os comandos disponíveis, atalhos e dicas de uso do Claude Code.
/clear
Limpar Contexto Reinicia a conversa atual sem perder as memórias persistentes. Útil ao trocar de tarefa.
/compact
Compactar Contexto Resume o histórico da conversa para liberar espaço de contexto sem perder o fio da meada.
/commit
Criar Commit (skill) Analisa as mudanças staged, gera uma mensagem de commit descritiva e realiza o commit.
/review-pr
Revisar Pull Request (skill) Revisa um PR existente no GitHub, listando problemas, sugestões e pontos de atenção.
/fast
Modo Rápido Alterna o modo de saída rápida. Usa o mesmo modelo, mas otimiza para velocidade de resposta.
/schedule
Agendar Tarefas (skill) Cria, lista ou executa agentes remotos agendados com cron. Automação de tarefas recorrentes.
/loop [intervalo]
Repetição Periódica (skill) Executa um comando em intervalos regulares. Ex: /loop 5m /commit commita a cada 5 minutos.
/simplify
Simplificar Código (skill) Revisa o código alterado buscando reutilização, qualidade e eficiência, corrigindo os problemas encontrados.
/pdf
Manipular PDFs (skill) Lê, cria, combina, divide ou adiciona marcas d'água em arquivos PDF.
/xlsx
Planilhas Excel (skill) Abre, edita, cria ou corrige arquivos .xlsx, .xlsm e .csv.
/docx
Documentos Word (skill) Cria, lê ou edita arquivos .docx com formatação, tabelas e estilos profissionais.
/pptx
Apresentações PowerPoint (skill) Cria ou extrai conteúdo de arquivos .pptx (slides).

Sistema de Memória

O Claude Code possui memória persistente entre conversas. Há 4 tipos de memória — cada uma com propósito específico.

user

Informações sobre quem é o usuário: cargo, experiência, linguagens que domina, preferências de comunicação.

feedback

Correções e confirmações de abordagem. O que não fazer e o que funcionou bem. Evita repetir erros.

project

Contexto do projeto em andamento: decisões, prazos, motivações por trás das tarefas. Inclui datas absolutas.

reference

Links para recursos externos: dashboards, boards de tarefas, canais Slack, docs específicos do projeto.

Como pedir ao Claude para lembrar algo "Claude, lembre-se que usamos PostgreSQL neste projeto, nunca SQLite." — o Claude vai salvar isso como memória de projeto automaticamente.
Como pedir ao Claude para esquecer algo "Claude, esqueça o que você salvou sobre o prazo do projeto." — o Claude vai localizar e remover a memória correspondente.
O que NÃO é salvo em memória Padrões de código, arquivos do projeto, histórico git, soluções de debug — essas coisas já estão no código. A memória é para o que não está no código.

O Arquivo CLAUDE.md

O CLAUDE.md é carregado automaticamente no início de toda conversa. É onde você define as regras permanentes do seu projeto.

# CLAUDE.md — exemplo para projeto web

# Contexto do Projeto
Sistema de gestão de clientes desenvolvido em Laravel + Vue 3.
Banco de dados: PostgreSQL 15. Deploy na Render.

# Convenções de Código
- PHP: PSR-12, nomes em inglês, métodos em camelCase
- Vue: Composition API (não Options API), TypeScript
- Commits: padrão Conventional Commits (feat/fix/docs/refactor)
- Nunca usar console.log em produção

# Comandos Importantes
- Testes: php artisan test
- Dev local: npm run dev && php artisan serve
- Lint: ./vendor/bin/pint && npm run lint

# Regras
- Sempre perguntar antes de deletar arquivos
- Não instalar pacotes sem aprovação
- Manter migrations compatíveis com rollback
- Não gerar arquivos README.md automaticamente
Onde criar

Na raiz do repositório (./CLAUDE.md). Também pode existir um global em ~/.claude/CLAUDE.md para regras que valem para todos os projetos.

O que incluir

Contexto do projeto, stack tecnológica, comandos de build/test, convenções de nomes, restrições de segurança e regras específicas da equipe.

O que NÃO incluir

Código duplicado do projeto, senhas ou tokens (nunca!), informações que mudam frequentemente sem atualização correspondente.

Hooks — Automação de Comportamentos

Hooks são comandos shell que executam automaticamente em resposta a eventos do Claude Code. Configurados em .claude/settings.json.

// .claude/settings.json — exemplo de hooks
{
  "hooks": {
    // executa ANTES de qualquer ferramenta ser usada
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "echo Bash sobre to run" }]
      }
    ],
    // executa APÓS qualquer ferramenta terminar
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "npm run lint -- --fix" }]
      }
    ],
    // executa quando o usuário envia uma mensagem
    "UserPromptSubmit": []
  }
}
PreToolUse

Executa antes de uma ferramenta ser usada. Pode bloquear a ferramenta retornando um código de saída não-zero. Útil para validações.

PostToolUse

Executa após uma ferramenta completar. Ideal para rodar linters, formatadores ou testes automaticamente após edições de arquivo.

UserPromptSubmit

Executa quando o usuário envia uma mensagem. Útil para logging, notificações ou preparar contexto adicional.

Boas Práticas

1

Sempre crie um CLAUDE.md antes de começar

Documente o stack, convenções e restrições do projeto. Isso evita que o Claude tome decisões erradas por falta de contexto.

2

Seja específico nas suas solicitações

Em vez de "conserte o bug", diga "no arquivo src/auth/login.js linha 47, o token não está sendo validado corretamente quando o usuário usa OAuth". Contexto preciso = melhor resultado.

3

Use /clear ao trocar de tarefa

O contexto da conversa anterior pode confundir o Claude em tarefas não relacionadas. Limpar o contexto melhora o foco e a qualidade das respostas.

4

Revise as mudanças antes de aprovar

O Claude mostra as alterações para aprovação. Leia o diff antes de confirmar, especialmente para operações destrutivas (deletar arquivos, force push, reset).

5

Nunca guarde segredos em CLAUDE.md

Senhas, tokens, chaves de API — nada disso deve estar no CLAUDE.md ou em arquivos de memória. Use variáveis de ambiente e arquivos .env (no .gitignore).

6

Peça planejamento antes de implementação complexa

Para tarefas grandes, peça primeiro "me mostre um plano de como você faria isso" antes de pedir para implementar. Isso evita retrabalho em caso de discordância de abordagem.

7

Mantenha o CLAUDE.md atualizado

Quando o stack ou as convenções do projeto mudam, atualize o CLAUDE.md. Informações desatualizadas levam a sugestões erradas.

8

Use o modo de permissão adequado

Default: pede aprovação para operações arriscadas. Auto-approve: para fluxos de trabalho rápidos onde você confia no Claude. Comece com Default até se familiarizar.

9

Dê feedback quando algo estiver errado

"Não, não faça assim porque..." ensina o Claude e ele salva isso como memória de feedback. Da próxima vez, ele lembrará da sua preferência automaticamente.

10

Use agentes especializados para tarefas paralelas

O Claude pode lançar sub-agentes para pesquisa, exploração de código e execução de tarefas em paralelo. Solicite isso explicitamente para tarefas independentes grandes.

⚡

Dicas Rápidas

Peça explicações no seu nível "Explique isso como se eu fosse iniciante em Python" ou "Explique de forma técnica, já sei como funciona o framework." O Claude adapta a linguagem ao seu perfil.
Referencie arquivos pelo caminho completo Em vez de "o arquivo de login", diga "src/auth/LoginController.php". Isso evita ambiguidade em projetos com muitos arquivos.
Use o formato arquivo:linha para navegar O Claude retorna referências no formato arquivo.py:42 — clicável direto no editor. Peça ao Claude para usar esse formato quando apontar problemas.
Para tarefas de segurança, forneça contexto claro "Estou em um CTF", "estou fazendo pentest autorizado", "quero entender como esse ataque funciona para me defender." O contexto permite que o Claude ajude adequadamente.
Atalhos de teclado padrão no CLI
Enter → envia mensagem  |  Shift+Enter → nova linha  |  Ctrl+C → interrompe geração  |  Ctrl+R → busca no histórico
O Claude pode abrir um navegador para você Com o MCP Claude in Chrome habilitado, o Claude pode navegar em páginas web, preencher formulários e capturar screenshots durante a sessão.
Peça ao Claude para se identificar antes de agir "Antes de editar qualquer arquivo, me diga quais arquivos você vai modificar e por quê." Isso ajuda a revisar a abordagem antes da execução.
Modos de permissão disponíveis
default → aprova caso a caso  |  acceptEdits → auto-aprova edições  |  bypassPermissions → aprova tudo (cuidado!)
